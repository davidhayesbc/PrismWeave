🔧 GitHub Configuration

⚠️ Required Setup
You'll need a GitHub personal access token to save captured content to your repository.
1

Create Token

Go to GitHub Settings → Developer settings → Personal access tokens → Generate new token

2

Set Permissions

Select "repo" scope for full repository access (or "public_repo" for public repos only)

3

Copy Token

Copy the generated token and keep it secure - you'll enter it when first using the bookmarklet

4

Configure Repository

Specify your repository in "owner/repo" format where captured content will be saved

🛠️ Troubleshooting

Common Issues:

❌ Bookmarklet doesn't work

Solution: Some websites block external scripts. Try refreshing the page and clicking again, or use the browser extension instead.

🔐 GitHub API errors

Solution: Check that your personal access token is valid and has the correct repository permissions.

📄 No content captured

Solution: The page structure may be unusual. Try different content extraction settings or capture specific sections manually.

📏 File too large

Solution: GitHub has file size limits. Enable content truncation or capture smaller sections of the page.
